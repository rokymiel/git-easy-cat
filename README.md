# PROD–hack | воркшоп по GIT

## Шаг 1 – Создание свежей ветки

Переключитесь на ветку `feature-leagcy-branch`. Тут располагается недописанная фича. Для этого можете использовать `git checkout feature-leagcy-branch`
Создайте с нее ветку `feature/<ВАШ ЛОГИН>`. Можно использовать команду `git checkout -b feature/<ВАШ ЛОГИН>`

Сначала поправьте последний коммит. Для этого добавьте следующие изменения , а после используйте `git commit --amend` (или `git reset --soft HEAD^ && git add file_path & git commit`)

```
    cat = r"""
 /\_/\      /\–/\      /\_/\
( o.o )    |.>ω<.|    ฅ •ﻌ• ฅ
 > ^ <      \___/      \___/
"""
```

По желанию можете добавить какой-то комит с котиками)) Например картинку положить

## Шаг 2 – Подтягиваем хотфиксы
В ветке `hotfixes` находятся правки критичкских ошибок, которые еще не влили в main. Вам нужны исправления из коммита `19d9c27e72599af840c2dc002db9426e53d12e47` (Исправлены константы цветов)

Перенесите изменения данного коммита в свою ветку. Используйте команду `git cherry-pick КОММИТ_ФИКСА` 

## Шаг 3 – Вливаем фичи
В ветке `feature-loop` находится недописанная фича, которую решили вливать в рамках вашей задачи

Поэтому нам надо влить изменения из ветки `feature-loop` командой `git merge feature-loop`

## Шаг 4 – Подтягиваем main

Сейчас будем отправлять наши изменения на ревью. Для этого подтягиваем изменения, которые были в main'е. Используйте команду `git rebase main`

После этого используйте команду `git push origin feature/<ВАШ ЛОГИН> `
